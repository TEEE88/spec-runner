# 業務ロジック概要

主要なビジネスロジック・計算式・制約を整理する。詳細な実装仕様は詳細設計に書く。

```yaml
ユースケース一覧:
  - id: UC-01
    名前: "{ユースケース名}"
    us_id: US-01
    概要: "{誰が何をするか}"

主要計算ロジック:
  - name: "{ロジック名}"
    入力: "{入力値・条件}"
    処理: "{計算・判定内容}"
    出力: "{出力値・結果}"

ビジネス制約:
  - name: "{制約名}"
    対象: "{集約・エンティティ}"
    内容: "{制約条件}"
```

## ユーザーストーリー

ユーザーストーリーは「{誰として} {何をしたい}」の形で書く。
UC との対応は yaml フェンスの `ユースケース一覧` が正本（scan.js が node_id 実在チェック）。

### US-01: {ストーリータイトル}

{誰として}、{何をしたい}。
それにより、{何を得たい}。
