Solution

整合性を
不変条件にする

「設計書とコードが常に一致」を機械的に強制する
docs 正本は手段。目的は整合性の維持。

spec-runner はこの不変条件を強制するハーネスである

github.com/TEEE88/spec-runner spec-runner
How it works

5つの強制機構

maps_to設計書とコードの対応を宣言する
scan.js + hooksEdit/Write のたびに実在を検証。なければ exit 1
@design-reviewer責務・入出力の意味的整合をエージェントが検証
design-changeフェーズゲートでコードを先に変えることを防ぐ
depends_on変更の波及範囲を依存関係として宣言する
