Core Concept

整合性を
不変条件にする

docs 正本（設計書を先に書く）は手段であって目的ではない。
目的は「設計書とコードが常に一致していること」の維持。
整合性が崩れた瞬間、AI は推測で動き始める。

Enforcement

不変条件の強制機構

maps_to設計書とコードの対応を宣言する。パス推測禁止
scan.js + hooksmaps_to の実在を Edit/Write のたびに検証。なければ exit 1
@design-reviewer責務・入出力・テスト観点の意味的整合をエージェントが検証
design-changeフェーズゲートでコードを先に変えることを構造的に防ぐ
depends_on変更の波及範囲を設計書の依存関係として宣言する
Declaration

frontmatter が
整合性の宣言

maps_to が設計書とコードをつなぐ唯一の正本。scan.js がこの宣言を検証し続ける。

---
spec_runner:
  node_id: detail.src.agents.reconciliation.agent
  kind: detailed_design
  depends_on:
    - overview.system_context
  maps_to:
    - src/agents/reconciliation/agent.py
    - tests/agents/reconciliation/test_agent.py
---
maps_to が腐ると
scan.js が exit 1 を返す
design-change では
depends_on を辿り波及範囲を特定
