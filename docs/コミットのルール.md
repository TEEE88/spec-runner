# コミットメッセージルール

このリポジトリでは Conventional Commits をベースに、件名は日本語で記述する。

## 形式

```text
<type>: <summary>

<body (任意)>
```

## type の使い分け

- `feat`: 新機能・新規ファイル追加
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `refactor`: リファクタリング（機能変更なし）
- `test`: テストの追加・修正
- `chore`: ビルド・設定・CI などの変更
- `style`: 動作に影響しない整形・命名変更

## summary のルール

- 日本語で書く
- 末尾に句点（。）を付けない
- 50 文字以内を目安にする
- 命令形を避け、「〜を追加」「〜を修正」などで記述する

## body のルール（任意）

- 変更が大きい場合のみ記載する
- 「何を」ではなく「なぜ」を書く

## 例

```text
feat: スライドテンプレートを追加
```

```text
fix: インストーラーの引数解析不備を修正
```

```text
docs: README にセットアップ手順を追記
```

```text
chore: プロジェクト初期構成を追加
```

## このリポジトリの初期コミット推奨

```text
chore: プロジェクト初期構成を追加
```

## 運用ルール

- 1 コミット 1 論理変更を原則とする
- ステージングはファイルを明示して行う
- コミット前に機密情報（API キー、トークン、秘密鍵など）を確認する
