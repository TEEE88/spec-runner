AI 駆動開発の現実

速く作れる。でも保守できない。

AI は動くコードをすぐ生成する。でも後から触れないコードを量産しているだけかもしれない。

この問題を解決しようと、2ヶ月かけてライブラリを作った。

github.com/TEEE88/spec-runner
課題 1 / 3

ドキュメントがない → 保守不能

📄 ドキュメントを作らないので、修正のたびにコードを全部調査しないといけない 「このコードが何をしているか」を毎回ゼロから理解する
🌀 完全 AI 任せだとアーキテクチャが崩壊する 共通化なし・ハードコード多発・ロジック分散・責務が適当 → 後になるほど修正コストが増大
🤔 なぜこの設計にしたのかが記録されない 後から「なぜこうなっているか」が誰にもわからない
github.com/TEEE88/spec-runner
課題 2 / 3

AI の速さが影響範囲を隠す

💥 何かを修正したときに影響範囲がわからない 「実際に動かして確認する」しかない → どこかでエラーが起きていることが多い
🔍 影響範囲の調査に時間がかかる ドキュメントやコードを一から全部読み込んでいく → AI も同じく全部読む
🤖 AI に毎回前提を説明しないといけない 「このシステムはこういう設計で…」を毎セッション繰り返す
github.com/TEEE88/spec-runner
課題 3 / 3

既存ツールは使いにくい

⌨️ コマンドを覚えて自分で打たないといけない 仕様駆動開発のライブラリは、手動でコマンドを選ぶ手間がある
🔤 生成されるドキュメントが英語ファイル名・詳細すぎる 日本語チームには読みにくい。内容が多すぎて見切れない
🧪 テストを後から書いても「既存挙動の確認」にしかならない 先に書かないとバグを検出する力がない

課題まとめ

AI 駆動開発の7つの課題

✕ドキュメントがなく保守不能
✕影響範囲がわからない
✕アーキテクチャが崩壊する
✕AI に毎回前提を説明する
✕設計判断が記録されない
✕コマンドを覚えないといけない
✕テストを後から書いても意味がない

これを解決するために 2ヶ月かけて spec-runner を作った

github.com/TEEE88/spec-runner
解決 1 — コマンドを打たせない

自動で最適なフローを選ぶ

AUTO Claude Code 起動時に docs/ の有無を判定して自動起動 新規プロジェクト → architecture-definition を起動 / 既存 → existing-project-to-docs を起動
SKILL 状況に応じて最適なスキルを AI が選ぶ ユーザーは「何を作りたいか」だけ言えばいい。コマンドを覚える必要はない
ARCH architecture.yaml でアーキテクチャ設計契約を記録 DDD / レイヤード / プラグイン型 → それぞれ専用スキルを自動生成
github.com/TEEE88/spec-runner
解決 2 — ドキュメント設計

日本語・階層・テンプレート

構造 要件定義 → 概要設計 → 詳細設計の3層。上から順に読めば全体がわかる
日本語 ファイル名・フォルダ名を日本語に統一。チームが迷わない
粒度 ドキュメントにコードを書かない。振る舞いだけを記述する 
docs/
├── 01_要件定義/
│   └── 要件定義.md
├── 02_概要設計/
│   ├── ユースケース一覧.md
│   ├── システム全体俯瞰.md
│   └── 90_ADR/       ← 設計判断を記録
└── 03_詳細設計/
    └── src/**         ← src/ をミラー
github.com/TEEE88/spec-runner
解決 3 — 整合性駆動開発

ドキュメントとコードを常に一対に

maps_to 設計書 ↔ コードの対応を宣言する
scan.js + hooks Edit/Write のたびに実在を検証。なければ exit 1
@design-reviewer 責務・入出力の意味的整合をエージェントが検証
TDD スキル テストを先に書くことを強制する

設計書が腐ったら
自動で検知する。

コードを先に変えることを
構造的に防ぐ。

github.com/TEEE88/spec-runner
解決 4 — 影響範囲の見える化

このUCを修正するとどこに影響するか、即わかる

宣言 depends_on に依存する設計書を書く
自動 scan.js が graph.json に依存グラフを自動生成
追跡 design-change でこのグラフを辿り影響ドキュメント・コードを列挙 
// graph.json（自動生成）
{
  "nodes": {
    "detail.usecase.注文確定": {
      "depends_on": [
        "overview.use_case_list",
        "detail.domain.注文"
      ],
      "maps_to": [
        "src/order/confirm.py",
        "tests/order/test_confirm.py"
      ]
    }
  },
  "reverse_index": { ... } ← 上流を即追跡
}
